COSAS a recordar:**

  • Para iniciar el blog:

  • En RStudio crea un nuevo proyecto (Web Site with Hugo). Has de poner el theme, yo he puesto yoshiharuyamashita/blackburn pero tiene pbs con los htmlwidget, así que pruebo el theme “hugo-tranquilpeak-theme” que lo has de poner como kakawait/hugo-tranquilpeak-theme. Se lo ví a https://aurora-mareviv.github.io/; pero tampoco me solucionaba el problema, así que sigo con el blacburn theme.

  • Si quieres que el deploy sea en ./docs de Github tienes que incluir en el config. toml: publishDir = "docs" ; y ademas incluir un archivo named .nojekyll in the root of your pages repo and pushing it to GitHub. This should only be necessary if your site uses files or directories that start with underscores since Jekyll considers these to be special resources and does not copy them to the final site.

  • El logo (el ababol) está en el config-toml en brand. La afoto está en ./static/img/logo.jpg, y el trozo de código está en . /layout/partials/brand.html. Me explico, eh!

  • Cuando haces el deploy en Netlify en las opciones de Build & Deploy has de (lo explica en blogdown):
    • Build command: lo he dejado en blanco, creo que podía poner hugo deploy
    • Publish directory: public (esta es la carpeta del repo de Github q tiene en realidad el blog)
    • Production branch: master (lo tengo en la master branch del repo)
